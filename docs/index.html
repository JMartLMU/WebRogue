WebRogue

WebRogue is a compact, statically checked scripting language for browser roguelike flow. It lets authors declare objects, define states, drive dialogue, create choices, and compile the result to clean JavaScript.

About WebRogue

WebRogue started as a compiler-construction project for browser game scripting. It is designed to make simple roguelike-style scenes easier to author by checking names, types, control flow, state transitions, and dialogue before the game ever runs.

The language blends a declarative page model with familiar imperative features. You can declare `object` and `state`, then use typed `let` bindings, `if` / `else`, `while`, `break`, `return`, functions, `_jump`, and `choice` to express game logic safely.

This site is a single-page companion for WebRogue: it tells the language story, showcases five complete example programs, explains the interesting static checks, and links directly back to the GitHub compiler repository.

Language coverage

Declarations

`object`, `state`, `function`, and `let` declarations are all checked for consistency before generation.

Control flow

`if` / `else`, `while`, `break`, `return`, and statically verified choice menus are all supported.

State & flow

States require `title`, `description`, and `contains`. `_jump` transitions are checked for valid state targets, and dialogue may be attached to states.

Example programs

Below are five complete WebRogue programs that exercise the language syntax and many of the interesting semantic checks.

1. Minimal start state

object Hero {
  name: string = "Ari";
  hp: number = 10;
  alive: boolean = true;
}

state Start {
  title: "A Quiet Camp";
  description: "A fire crackles while hunters prepare to leave.";
  contains: [Hero];
}

_jump(Start);

2. Dialogue and choice

object Guide {
  name: string = "Ena";
  hp: number = 8;
  alive: boolean = true;
}

state Entrance {
  title: "Mossy Gate";
  description: "A lantern hangs from twisted iron.";
  contains: [Guide];
  dialogue(keyboard.space, "Press space to continue", "You stand before the gate.", "Ena whispers, 'Choose carefully.'");
}

state Decision {
  title: "Crossroads";
  description: "Three paths fan out under the pines.";
  contains: [Guide];
}

_jump(Entrance);
choice(num, Forward, Back) {
  option Forward { _jump(Decision); }
  option Back { print "You turn back to safety."; }
}

3. Typed function with return

function heal(amount: number) -> number {
  if amount < 0 {
    return 0;
  } else {
    return amount + 5;
  }
}

let currentHp: number = 3;
currentHp = heal(currentHp);

if currentHp > 7 {
  print "Ready for the fight.";
} else {
  print "Weakened but steady.";
}

4. Loop with break

let count = 0;

while count < 5 {
  count = count + 1;
  if count == 3 {
    print "Third step reached.";
    break;
  }
}

print "Final count is " + count;

5. State flow with objects and inference

object Potion {
  name: string = "Potion";
  hp: number = 4;
  alive: boolean = true;
}

state Chamber {
  title: "Crystal Chamber";
  description: "Blue light washes over a stone dais.";
  contains: [Potion];
}

let nextState = Chamber;
_jump(nextState);

Developer

Jacob Martinez built WebRogue as a compiler project that brings static checking to browser game scripting. The language is intentionally small but strong enough to catch broken state transitions, bad types, invalid choice menus, and non-returning functions before generation.

Jacob's work focuses on making program structure visible in a way that supports both author productivity and compiler correctness.

Find the compiler source on GitHub: github.com/JMartLMU/WebRogue.
