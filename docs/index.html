WebRogue

A compact statically checked language for browser roguelike scripts: declare objects, wire states, jump between pages, and compile the result to readable JavaScript.

Read the language guide Open an example GitHub repository

Language Shape

Game declarations

Objects compile to JavaScript objects. States require titles, descriptions, and object references.

Static checks

Types, scopes, returns, breaks, calls, duplicate names, state contents, and jumps are checked before generation.

Compiler phases

The pipeline follows the Carlos model: parse, analyze, optimize, generate JavaScript, then optionally run it from the CLI.

Example

object Player {
  name: string = "Mira";
  hp: number = 12;
}

state Start {
  title: "Dungeon Entrance";
  description: "Cold air leaks from the stone archway.";
  contains: [Player];
}

_jump(Start);
print "The first state is ready.";

Implemented Scope

Working

Primitive types, functions, control flow, objects, states, checked jumps, static analysis, optimization, and JS output.

Not implemented

General arrays, object literals, field access, imports, browser runtime helpers, and a game engine runtime.
