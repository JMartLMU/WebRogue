WebRogue

WebRogue is a compact, statically checked scripting language for browser roguelike flow. It lets authors declare objects, define states, drive dialogue, create choices, and compile the result to clean JavaScript.

About WebRogue

WebRogue was inspired by the difficulty of creating games as a begginner programmer using pygame in the Intro to Programming class at LMU. My difficulty with scripting and sharing led me to shape WebRogue into a lightweight language with features to mimmic a small game engine.

WebRogue started as a compiler-construction project for browser game scripting. It is designed to make simple roguelike-style scenes easier to author by checking names, types, control flow, state transitions, and dialogue before the game ever runs.

The language blends a declarative page model with familiar imperative features. You can declare `object` and `state`, then use typed `let` bindings, `if` / `else`, `while`, `break`, `return`, functions, `_jump`, and `choice` to express game logic safely.

This site is a single-page companion for WebRogue: it tells the language story, showcases five complete example programs, explains the interesting static checks, and links directly back to the GitHub compiler repository.

Language coverage

Declarations

`object`, `state`, `function`, and `let` declarations are all checked for consistency before generation.

Control flow

`if` / `else`, `while`, `break`, `return`, and statically verified choice menus are all supported.

State & flow

States require `title`, `description`, and `contains`. `_jump` transitions are checked for valid state targets, and dialogue may be attached to states.

3. Functions

function isAlive(hp: number) -> boolean {
  return hp > 0;
}

function describe(name: string, hp: number) -> string {
  if isAlive(hp) {
    return name;
  } else {
    return "fallen";
  }
}

let label: string = describe("Mira", 10);
print label;

4. Choice Menu

let playerHp: number = 10;
let enemyHp: number = 6;

print "Choose an action:";

choice(num, Attack, Heal) {
  option Attack {
    enemyHp = enemyHp - 2;
    print "Attacked.";
    print enemyHp;
  }

  option Heal {
    playerHp = playerHp + 2;
    print "Healed.";

8. Fibonacci

function fib(n: number) -> number {
  if n <= 1 {
    return n;
  } else {
    return fib(n - 1) + fib(n - 2);
  }
}

let count: number = 10;
let i: number = 0;

while i < count {

10. Tiny Dungeon

object Player {
  name: string = "Mira";
  hp: number = 12;
  alive: boolean = true;
}

object Slime {
  name: string = "Slime";
  hp: number = 4;
  alive: boolean = true;
}

state Start {
  title: "Dungeon Start";
  description: "A half-burned torch marks the safe edge of the dungeon.";
  contains: [Player];
}

state Entry {
  title: "Mossy Entry";
  description: "Cold stairs descend into the first pocket of dungeon air.";
  contains: [Player, Slime];
}

function damage(hp: number, amount: number) -> number {
  return hp - amount;
}

let playerHp: number = 12;
if playerHp > 0 {
  print "ready";
} else {
  print "fallen";
}

while playerHp > 9 {
  playerHp = damage(playerHp, 1);
  if playerHp == 10 {
    break;
  }
}

_jump(Entry);
print "entered Entry";
print playerHp;

Developer

My name is Jacob Martinez and I built WebRogue as a compiler project that brings static checking to browser game scripting. The language is intentionally small but strong enough to catch broken state transitions, bad types, invalid choice menus, and non-returning functions before generation.

My work focuses on making program structure visible in a way that makes programming more accessible for begginer game designers specifically with a lightweight approach.

I worked on this project as an assignemnt for my coursework. If I had more time, I would have liked to add a rendering feature to more mimick a lightweight game engine.

Find the compiler source on GitHub: github.com/JMartLMU/WebRogue.
