WebRogue

A compact statically checked language for browser roguelike scripts: declare entities, wire rooms, write turn logic, and compile the result to readable JavaScript.

Read the language guide Open an example GitHub repository

Language Shape

Game declarations

Entities compile to JavaScript objects. Rooms require titles, descriptions, and entity references.

Static checks

Types, scopes, returns, breaks, calls, duplicate names, and room contents are checked before generation.

Compiler phases

The pipeline follows the Carlos model: parse, analyze, optimize, then generate JavaScript.

Example

entity Player {
  name: string = "Mira";
  hp: number = 12;
}

room Entrance {
  title: "Dungeon Entrance";
  description: "Cold air leaks from the stone archway.";
  contains: [Player];
}

print "Two rooms are ready.";

Implemented Scope

Working

Primitive types, functions, control flow, entities, rooms, static analysis, optimization, and JS output.

Not implemented

General arrays, object literals, field access, imports, browser runtime helpers, and a game engine runtime.
